Overview
This repo contains the complete infrastructure for the Prompt2Test platform — an AI-powered test case generation and browser automation tool. Everything is defined as code using AWS CDK (TypeScript). A single command (./deploy.sh) deploys the entire platform to a new AWS account — VPC, databases, Lambda, ECS, AI agent, CI/CD pipelines, and the React UI hosting.
TL;DR for a new account: Fill in cdk.json, run ./deploy.sh, follow 3 browser prompts. Done.

Deploying to a New AWS Account
Shortcut: Run ./deploy.sh — it handles everything automatically and only pauses 3 times for browser steps.
  • 1
    Verify prerequisites 
    aws --version              # must be v2
node --version             # must be v18+
cdk --version              # must be v2.x
aws sts get-caller-identity  # confirm credentials
  • 2
    Enable Bedrock model access (browser)

    AWS Console → Amazon Bedrock → Model access → Manage model access
    Enable: Claude Sonnet 4.5 (Anthropic) and Titan Embed Text v2 (Amazon)

  • 3
    Create GitHub CodeStar Connection (browser)

    AWS Console → Developer Tools → Settings → Connections → Create connection → GitHub
    Name it Prompt2TestGitHub, authorize, connect. Copy the Connection ARN.

  • 4
    Update cdk.json with your values 
    {
  "context": {
    "githubOwner": "your-github-username",
    "githubConnectionArn": "arn:aws:codeconnections:us-east-1:ACCOUNT:connection/UUID"
  }
}
  • 5
    Run the deploy script 
    npm install
cdk bootstrap aws://<ACCOUNT_ID>/us-east-1
./deploy.sh

    The script will: deploy CDK stack → trigger 3 pipelines → create AgentCore runtime → set Amplify env vars → create admin user. All output values saved to deployment-outputs.txt.

  • 6
    Connect Amplify to GitHub (browser)

    AWS Console → Amplify → Prompt2TestUI → Connect branch → GitHub → Prompt2TestUI repo → branch: master → Save and deploy

  • 7
    Add teams and services (UI)

    Log in as admin → Config tab → create team groups, add users, add services and accounts. All config stored in DynamoDB — no CLI needed.

Post-Deploy: Adding Teams & Services
After deploy, all configuration is done through the UI Config tab — no CLI needed. Everything is stored in DynamoDB (prompt2test-config).
TaskWhereDetail
Create team groupsConfig tab → TeamsCreate a group (e.g. teama) — maps to Cognito group
Add users to teamsConfig tab → UsersAssign user to group; team resolved from JWT cognito:groups claim
Add servicesConfig tab → ServicesAdd service name + URL per environment (dev/qa/uat/prod)
Add test accountsConfig tab → AccountsAdd account credentials per environment for browser tests
Key Design Decisions
DecisionChoiceReason
Config storageDynamoDB not SSMBrowser can call DynamoDB directly via Cognito IAM; SSM requires server-side proxy
Team membershipCognito GroupsJWT cognito:groups claim — no extra DB lookup needed at runtime
Browser sessionsECS Fargate on-demandOne task per test run, stopped after completion — zero idle cost
Agent runtimeBedrock AgentCoreManaged container runtime — no server to maintain, scales automatically
Test case storageAurora Serverless v2 + pgvectorSemantic search via vector embeddings for duplicate detection
Full decision log: Prompt2Test_ArchitectureDecisions.html
Useful CLI Commands
# Check CDK stack outputs
aws cloudformation describe-stacks --stack-name Prompt2TestStack \
  --query "Stacks[0].Outputs" --region us-east-1

# Check pipeline status
aws codepipeline list-pipeline-executions \
  --pipeline-name prompt2test-agent-pipeline --max-results 1

# Check AgentCore runtime status
aws bedrock-agentcore-control list-agent-runtimes --region us-east-1

# Check running ECS tasks
aws ecs list-tasks --cluster prompt2test-playwright-cluster --region us-east-1

# View Lambda logs (live tail)
aws logs tail /aws/lambda/p2t-testcase-reader --follow --region us-east-1
aws logs tail /aws/lambda/p2t-testcase-writer --follow --region us-east-1

# Manually trigger all 3 pipelines
for P in prompt2test-lambda prompt2test-agent-pipeline prompt2test-playwright-mcp-pipeline; do
  aws codepipeline start-pipeline-execution --name $P --region us-east-1
done
