Request Flow — Step by Step

1

User Opens Browser

Tester navigates to the Amplify-hosted React SPA via CloudFront CDN. Cognito handles sign-in and issues JWT tokens with group claims (admin/viewer).

2

Prompt Submitted

User enters a natural language test prompt. The React app calls Bedrock AgentCore via the Cognito IAM auth role — no backend middleware needed.

3

AI Processes Request

Bedrock AgentCore runs Claude Sonnet 4.5 with Strands SDK tools. The agent decides which tools to invoke: execute_test, save_test_case, analyze_results.

4

Browser Automation

Agent calls ECS Fargate via MCP/WebSocket. Playwright MCP server controls a real Chromium browser. noVNC streams a live view back to the UI.

5

Test Persistence

Lambda function (p2t-testcase-fn) saves test cases to DynamoDB config table and Aurora Serverless v2. Screenshots go to S3. pgvector enables semantic search.

6

Results Returned

Agent streams structured results back through AgentCore → UI. User sees PASS/FAIL status, screenshots, and AI-generated analysis in the Execution Panel.

7

CI/CD Deployment

Any GitHub push triggers the relevant pipeline: Amplify auto-builds the UI, CodePipeline builds and deploys Lambda, Agent container, and Playwright MCP to ECS Fargate.

8

Security & Config

Cognito auth role grants least-privilege access. Secrets Manager holds Aurora credentials. SSM Parameter Store holds Aurora connection config. CloudWatch captures all logs and metrics.
