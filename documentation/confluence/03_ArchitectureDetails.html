2. Agent Backend (Bedrock AgentCore)

AWS Bedrock AgentCore

Runtime: ID: Prompt2TestAgent-YTVbD4GrTi, Version: 39 (current), Status: READY, Network: PUBLIC

Container: ECR: prompt2test-agent, Base: Python 3.12, Framework: Strands Agents, Port: 8080 (internal)

LLM: Claude Sonnet 4.5, us.anthropic.claude-sonnet-4-5-20250929-v1:0, Cross-region inference

Agent Modes (main.py):

  • plan_scenario -- enrich test scenario with real config from DynamoDB (SERVICE#{team}#{env})
  • start_session -- RunTask → poll RUNNING → get public IP via ENI
  • automate -- LLM drives Playwright via MCP tools → captures playwright_calls per step → StopTask
  • replay -- deterministic re-run using saved playwright_calls (raw ClientSession.call_tool, no LLM)
  • stop_session -- explicit ECS StopTask cleanup (called on Stop, success, or failure)

plan_scenario receives env + team in payload → agent reads DynamoDB SERVICE#{team}#{env} partition for service config when generating the plan

IAM Role

prompt2test-agentcore-role

Permissions: bedrock:InvokeModel, bedrock:ConverseStream, ssm:GetParameter, dynamodb:Query / dynamodb:GetItem, ecs:RunTask / DescribeTasks / StopTask, ec2:DescribeNetworkInterfaces, iam:PassRole (task execution role), lambda:InvokeFunction, ecr:GetAuthToken, logs:*

DynamoDB -- prompt2test-config

Service Config (UI + Agent reads/writes): pk=SERVICE#{team}#{env}, sk={svc}#{key}, val={value}. e.g. pk=SERVICE#teamA#dev, sk=MyService#URL

Test Accounts (UI reads/writes): pk=ACCOUNT#{env}, sk={id}#{NAME|CODE}, val={value}

Access: Browser → Cognito IAM creds, Agent → DynamoDBConfigRead policy, dynamodb:Query, dynamodb:GetItem

On-Demand Session Flow

  1. User describes test → Agent reads DynamoDB SERVICE#{team}#{env} → generates plan
  2. User selects service chip + TC name → Save Test Case → Lambda saves to Aurora + embeds vector
  3. User opens TestCasePage → clicks Automate → popup opens instantly with progress bar
  4. start_session: reads SSM config → RunTask (dedicated Fargate task)
  5. Poll DescribeTasks (~60s) → RUNNING → public IP via ENI → TCP poll port 3000 until ready
  6. Pre-warm: MCP connect → browser_navigate about:blank → Chromium open in noVNC
  7. automate: LLM drives Playwright via MCP → captures playwright_calls per step → streams results
  8. User saves steps → Save Steps persists playwright_calls to Aurora via Lambda
  9. Run Test: replay mode → raw ClientSession.call_tool() (no LLM) → StopTask → PASS/FAIL recorded

Flow: Session Resolution

Agent (start_session)→ SSM →RunTask → ENI IP→ TCP poll :3000 →Pre-warm browser→Return ip:3000 + ip:6080

5. Playwright MCP -- ECS Fargate (On-Demand per Session)

On-Demand Task Lifecycle

Startup (~60s): RunTask → PROVISIONING → PENDING → RUNNING. ENI assigns public IP, TCP poll :3000 until ready. Pre-warm: browser_navigate about:blank, Chromium open before noVNC loads.

During Test: 1 task = 1 user = 1 Xvfb display. Full screen isolation, No cross-user interference, No --isolated flag -- browser persists between MCP connections.

Teardown: StopTask called on test end. Task stopped in seconds. Pay only for test duration, ~10-15 min avg per session.

ECS Fargate -- playwright-mcp Tasks

Per-user isolation: Each user gets their own Fargate task.

  • User A's Task: RUNNING, Own public IP, Own Xvfb :99 display, MCP → ip-A:3000, noVNC → ip-A:6080
  • User B's Task: RUNNING, Own public IP, Own Xvfb :99 display, MCP → ip-B:3000, noVNC → ip-B:6080
  • No Warm Pool: 0 always-on tasks, No NLB (removed), No warm service, No autoscaling policy. Pure on-demand only.

Task Spec: CPU: 2 vCPU (ARM64 / Graviton), Memory: 4 GB, Image: ECR prompt2test-playwright-mcp:latest, Mode: BROWSER_MODE=headed (Xvfb + x11vnc + noVNC), Ports: 3000 (MCP) / 6080 (noVNC) / 8080 (health)

Cluster: prompt2test-playwright-cluster / VPC: prompt2test-playwright-vpc / Public subnets (2 AZs) / assignPublicIp: ENABLED

6. Scaling Model and Cost

On-Demand Scaling -- 1 Task per User Session

MetricValue
Always-on tasks (no warm pool)0
Tasks per active test session1
Concurrent users (Fargate limit)20+

Scale-Out: Automatic -- each start_session RunTask call = new task. No shared state, no contention. Unlimited concurrent users.

Scale-In: Automatic -- StopTask called when test completes or errors. Task goes STOPPED within seconds. Zero idle waste.

Isolation Model: Each user gets their own Fargate task with its own Xvfb display :99 -- no cross-user screen sharing. Users always see their own browser, never another user's test.
