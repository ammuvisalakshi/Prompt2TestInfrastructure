Overview

This repo contains the complete infrastructure for the Prompt2Test platform -- an AI-powered test case generation and browser automation tool. Everything is defined as code using AWS CDK (TypeScript). A single command (./deploy.sh) deploys the entire platform to a new AWS account -- VPC, databases, Lambda, ECS, AI agent, CI/CD pipelines, and the React UI hosting.

TL;DR for a new account: Fill in cdk.json, run ./deploy.sh, follow 3 browser prompts. Done.

Repository Structure

Prompt2TestInfrastructure/

  • bin/infrastructure.ts -- CDK entry point, reads cdk.json context, instantiates stack
  • lib/prompt2test-stack.ts -- ALL AWS resources defined here, one file, one stack
  • cdk.json -- CDK config, set githubOwner + githubConnectionArn before deploy

Deploying to a New AWS Account

Shortcut: Run ./deploy.sh -- it handles everything automatically and only pauses 3 times for browser steps.
  1. Verify prerequisites: aws --version (v2), node --version (v18+), cdk --version (v2.x), aws sts get-caller-identity
  2. Enable Bedrock model access (browser): AWS Console → Amazon Bedrock → Model access → Enable Claude Sonnet 4.5 and Titan Embed Text v2
  3. Create GitHub CodeStar Connection (browser): AWS Console → Developer Tools → Connections → Create connection → GitHub. Name it Prompt2TestGitHub. Copy Connection ARN.
  4. Update cdk.json: Set githubOwner and githubConnectionArn
  5. Run the deploy script: npm install → cdk bootstrap → ./deploy.sh. Outputs saved to deployment-outputs.txt.
  6. Connect Amplify to GitHub (browser): AWS Console → Amplify → Prompt2TestUI → Connect branch → master → Save and deploy
  7. Add teams and services (UI): Log in as admin → Config tab → create teams, add users, add services and accounts.

CI/CD Pipelines

prompt2test-agent-pipeline

Push to Prompt2TestAgent→CodeBuild→Docker build→ECR: prompt2test-agent

AgentCore picks up the new image on next invocation.

prompt2test-playwright-mcp-pipeline

Push to Prompt2TestPlaywrightMCP→CodeBuild→Docker build→ECR: prompt2test-playwright-mcp

ECS tasks pull the latest image on each test run.

prompt2test-lambda

Push to Prompt2TestLambda→CodeBuild→Zip artifact→lambda update-function-code x2

Updates both p2t-testcase-writer and p2t-testcase-reader simultaneously.

Amplify (UI)

Push to master (Prompt2TestUI)→Amplify detects push→npm build→Live in ~2 min

Post-Deploy: Adding Teams and Services

After deploy, all configuration is done through the UI Config tab -- no CLI needed. Everything is stored in DynamoDB (prompt2test-config).

Key Design Decisions

DecisionChoiceReason
Config storageDynamoDBBrowser can call DynamoDB directly via Cognito IAM; SSM requires server-side proxy
Team membershipCognito GroupsJWT cognito:groups claim -- no extra DB lookup
