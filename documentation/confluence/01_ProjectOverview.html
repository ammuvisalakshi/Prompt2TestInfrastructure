Prompt2Test -- Project Overview

An AI-powered test case generation and browser automation platform. QA engineers describe what they want to test in plain English -- the AI agent authors a test plan, spins up a live browser, and executes it end-to-end.

What It Does -- End to End Flow

QA Engineer (types a prompt)→ React UI (Amplify hosted)→ Bedrock AgentCore (Claude Sonnet 4.5)→ AI generates plan (structured steps)→ ECS Fargate task (spins up on demand)→ Playwright MCP (executes in browser)→ Live view noVNC (streamed to UI)→ PASS / FAIL (saved to Aurora)

Key Architecture Decisions

DecisionChoiceWhy
Config storageDynamoDBBrowser SDK calls DynamoDB directly via Cognito IAM -- no server proxy needed. SSM cannot be called from the browser.
Team membershipCognito GroupsTeam resolved from JWT cognito:groups claim at login -- zero extra lookups.
Browser sessionsECS Fargate on-demandOne task per test run, auto-stopped on completion. Zero idle cost -- pay only when tests run.
AI agent runtimeBedrock AgentCoreManaged container runtime -- no EC2/ECS service to maintain. Scales automatically.
Test case storageAurora + pgvectorVector embeddings enable semantic duplicate detection and similarity search.
InfrastructureAWS CDK (TypeScript)Full IaC -- entire platform deployable to a new AWS account with one command.
